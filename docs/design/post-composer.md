## 1. 概要・目的

盆栽の記録をテキスト・画像・ジャンルタグとともに投稿する画面。
新規投稿と既存投稿の編集の 2 パターンを 1 仕様で定義する（レイアウトは共通）。

- 新規投稿: FAB タップ → モーダルで表示（`posts/new`）
- 投稿編集: 投稿詳細の「⋮ メニュー → 編集」→ モーダルで表示（`posts/[id]/edit`）

### 1.1 解決する問題

- モバイルでの投稿作成導線を確立する（Web の `/posts/new` に相当する機能のモバイル版）
- 無料ユーザーとプレミアムユーザーの制約を画面上で明示し、アップグレードへの自然な導線とする

|------|------|
| 配置 | `posts/new` / `posts/[id]/edit`（タブ外スタック。モーダル表示） |

## 5. データの流れ

### 5.1 新規投稿

1. ユーザーが本文・ジャンル・画像を入力する
2. 「投稿する」タップ時、画像がある場合: presigned URL 取得 → R2 へ直接 PUT アップロード → URLをリクエストに含める
3. `POST /api/v1/posts`（想定エンドポイント。正本は OpenAPI）
4. 成功後: `posts.feed` / 自分の `users.detail` を `invalidateQueries`

### 5.2 投稿編集

1. 画面マウント時に既存投稿データ（`usePostQuery(id)`）を初期値としてフォームに展開する
2. 変更があった項目のみサーバーへ送る（フロントエンドの実装に委ねる）
3. `PATCH /api/v1/posts/{id}`（想定エンドポイント。正本は OpenAPI）
4. 成功後: `posts.detail(id)` / `posts.feed` を `invalidateQueries`

**core に要確認:** PATCH エンドポイントが部分更新（差分のみ送信）か全件置き換えかによってリクエスト設計が変わる。

### 5.3 画像アップロード

- `expo-image-picker` で端末内の画像を選択する
- アップロード前に `expo-image-manipulator` で圧縮する（品質・リサイズの具体的な設定値は `lib/constants/` の定数。**frontend への申し送り**）
- presigned URL 取得 → R2 に直接 PUT する（`api-client.md` 例外規定）

### 5.4 必要なデータ項目（編集初期値）

| 項目 | 意味 |
|------|------|
| `content` | 本文テキスト |
| `genres` | ジャンル配列（`{ id, name }[]`）|
| `media` | 既存メディア配列（`{ id, url, type, sortOrder }[]`）|

## 6. 本文入力エリア（PostBodyInput）の詳細仕様

### 6.1 テキストエリア

- `TextInput` の `multiline: true` / `scrollEnabled: true`（テキストが長くなった場合にエリア内でスクロール）
- 最小高さ: 120pt（3〜4 行分が最初から見える）
- 最大高さ: 制限なし（エリアは固定高さ。コンテナ全体が `ScrollView` でスクロール可能）
- `placeholder`: 「盆栽の記録をシェアしましょう...」
- `placeholderTextColor`: `colorTextTertiary`（`#8a8a8a`）
- フォント: `textBase`（14pt / lineHeight 22pt）/ `colorTextPrimary`
- 左右内側パディング: `spacing3`（12pt）/ 上下: `spacing3`（12pt）
- ボーダー: なし（テキストエリアはボーダーなしのフラットなデザイン。コンテナのセパレータで分割を示す）
- `keyboardType: "default"` / `autoCapitalize: "sentences"`（日本語入力に配慮）
- `returnKeyType: "default"`（改行を許容する）
- `accessibilityLabel`: 「投稿内容を入力」
- `accessibilityHint`: `isPremium` に応じた「最大500文字で入力してください」または「最大2000文字で入力してください」

### 6.2 文字数カウンタ

- 位置: テキストエリア右下（`textSm` / 12pt）
- 通常: `{現在文字数} / {上限}`（例: `123 / 500`）/ `colorTextSecondary`
- 残り 50 文字以下: `colorWarning`（`#b8860b`）に変化
- 上限超過（0 文字以下）: `colorError`（`#c0392b`）に変化
- 上限超過の場合、「投稿する」ボタンを disabled にする

### 6.3 上限超過時のエラー表示

- 文字数カウンタを `colorError` に変化させるのみ（インラインエラーテキストは出さない）
- 「投稿する」ボタンが disabled になるため、ユーザーはカウンタを見て削減する

## 7. ジャンル選択（GenreSelector）の詳細仕様

> **改訂注記（2026-07-12）:** 本セクションは実装 `components/post/GenreSelector.tsx` の確認結果に基づき全面改訂した。
>
> - **旧仕様（本文入力エリアの直下に全ジャンルチップを常時展開表示。見出しに「（最大3つまで選択）」という上限表記はあったが「（任意）」の表記が一切なく、実質必須の項目であるかのような見た目だった）は誤りだった。** 全文は §7.7 に取り消し注記として保存する。
> - **Web 本番（`Bon_Log_cfw` の `components/post/GenreSelector.tsx`）ではジャンル選択は常に任意である。** トリガーボタンのプレースホルダーに明示的に「ジャンルを選択（任意）」と表示し、1 つも選ばずに投稿できる（`maxSelections` 到達時のみ追加選択を disabled にする）。現行の Native 実装はこの Web 挙動に合わせて是正済みであり、本セクション（§7.1〜§7.6）はその実装を正として書き直したもの。
> - なお §10.4（送信ボタンの活性条件）は本改訂の前後を通じてジャンル選択数を条件に含めていない（本文 1 文字以上 OR 画像 1 枚以上のみが条件）。つまり送信そのものを妨げる必須項目ではなかったが、UI の見た目が実質必須であるかのような誤解を招く仕様だった、という点が今回の是正の要点である。

### 7.1 全体構成（トリガーボタン → モーダル）

ジャンルチップの一覧は常時表示しない。「ラベル」+「トリガーボタン」+（選択時のみ）「件数カウンタ」を本文入力エリアの下に縦積みし、トリガーボタンをタップしたときだけモーダルでチップ一覧を開く。

```
未選択時:
┌─────────────────────────────────────────────────────────────┐
│ ジャンル                                                     │
│ ┌─────────────────────────────────────────────────────┐  ▼ │
│ │ ジャンルを選択（任意）                                 │    │
│ └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘

選択時（例: 2件選択）:
┌─────────────────────────────────────────────────────────────┐
│ ジャンル                                                     │
│ ┌─────────────────────────────────────────────────────┐  ▼ │
│ │ 松柏類, 雑木類                                         │    │
│ └─────────────────────────────────────────────────────┘    │
│ 2/3 選択中                                                   │
└─────────────────────────────────────────────────────────────┘
```

- ラベル「ジャンル」: `textXs` / `colorTextTertiary` / `letterSpacingTight`。上限数はラベル自体には含めない（トリガーのプレースホルダーと件数カウンタで伝える）
- トリガーの表示テキスト: 未選択時はプレースホルダー「ジャンルを選択（任意）」。1 件以上選択時は選択中のジャンル名を `, ` 区切りで連結し、1 行で省略表示する（`numberOfLines={1}` / `ellipsizeMode="tail"`）
- 件数カウンタ「{選択数}/3 選択中」: トリガー直下、1 件以上選択している場合のみ表示する（0 件時は非表示）

### 7.2 チップの状態

トリガーからモーダルを開いた中で表示するジャンルチップ自体の見た目・状態は次の通り。**この配色・トークンは旧仕様から変更していない**（変わったのはチップの置き場所のみ）。`events.md` §2.1 の FilterBar チップ、`shops.md` §2 のジャンルチップは本表のスタイルを踏襲している。

| 状態 | 背景 | テキスト | ボーダー |
|------|------|---------|---------|
| 非選択 | `colorActionSecondary`（`#e9e9e9`）| `colorActionSecondaryText`（`#2e2e2e`）| 1pt `colorBorderLight` |
| 選択済み | `colorActionPrimary`（`#2e2e2e`）| `colorActionPrimaryText`（`#ffffff`）| なし |
| 無効（3つ選択済みで他チップ）| `colorSurfaceMuted`（`#f0f0f0`）| `colorTextTertiary`（`#8a8a8a`）| 1pt `colorBorderLight` |

- チップ角丸: `radiusSm`（6pt）/ 高さ 36pt
- テキスト: `textXs`（10pt）/ `letterSpacingTight`（0.5）
- 内側パディング: `paddingHorizontal: spacing3`（12pt）。上下は `justifyContent` / `alignItems` の中央揃えでセンタリングしており、固定の `paddingVertical` は持たない
- タップ領域: `hitSlop: { top: 4, bottom: 4, left: 4, right: 4 }` により、36pt の高さに対して実効タップ領域を 44pt（36 + 4 + 4）まで拡大する
- 選択済みチップは 3 件到達時でも常にタップ可能（解除できる）。disabled になるのは「未選択かつ 3 件到達済み」のチップのみ

**（2026-07-12 取り消し注記）:** 旧仕様の本表には「disabled（送信中）」という 4 行目と、「選択時アニメーション: 背景色切り替えを `durationFast`（200ms）でフェード」という記述があったが、実装にこれらは存在しない。送信中（`isDisabled=true`）はトリガーボタン自体が開かなくなるためモーダル内のチップ操作は発生せず、チップ単体に専用の「送信中」見た目は実装されていない。また状態切り替えに明示的なフェードアニメーションもなく、style の即時切り替えのみである。実装 `components/post/GenreSelector.tsx` に合わせて上表・上記箇条書きを修正した。

### 7.3 トリガーボタンの仕様

| 項目 | 値 |
|------|-----|
| 最小高さ | 44pt（タップターゲット確保）|
| 背景 | `colorBackground`（`#ffffff`）|
| ボーダー / 角丸 | 1pt `colorBorder`（`#c8c8c8`）/ `radiusSm`（6pt）|
| 内側パディング | 水平 `spacing3`（12pt）|
| テキスト | `textBase`（14pt）/ `colorTextPrimary`（`#1a1a1a`）/ 1 行省略 |

| accessibilityHint | 「タップしてジャンルを選択します」|
| accessibilityState | `{ disabled: isDisabled }` |

### 7.4 モーダルの構成と開閉

トリガーボタンをタップすると、画面下からスライドアップするモーダルシートでチップ一覧を表示する（`Modal` / `animationType="slide"` / `transparent` / `accessibilityViewIsModal`）。

```
┌─────────────────────────────────────────────────────────────┐
│ ░░░░░░░░░░░░░░░░ 背景スクリム（タップで閉じる） ░░░░░░░░░░░░ │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ ジャンルを選択                                    [×]   │ │ ← シートヘッダー
│ ├─────────────────────────────────────────────────────────┤ │
│ │ [松柏類] [雑木類] [草もの] [用品・道具]                  │ │
│ │ [施設・イベント] [その他]                                │ │ ← チップグリッド（flexWrap）
│ │                                                         │ │
│ │ 2/3 選択中                                              │ │ ← 1件以上選択時のみ
│ └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

| 要素 | 仕様 |
|------|------|
| 背景スクリム | `colorScrimLight`（`rgba(0, 0, 0, 0.40)`）/ タップでモーダルを閉じる |
| シート | 背景 `colorBackground` / 上端 2 隅のみ `radiusMd`（8pt）/ `shadowWashiLg` / 最大高さは画面の 70%（`maxHeight: "70%"`）/ 下端にセーフエリア分の余白（`insets.bottom`）を確保 |
| シートヘッダー | 背景 `colorSurfaceWashi`（`#f7f7f7`）/ 下ボーダー 1pt `colorBorderLight` / タイトル「ジャンルを選択」（`textLg` / `colorTextPrimary`）/ 右端に閉じるボタン（`close` アイコン 22pt・タップターゲット 44×44pt）|

| 閉じる手段 | ①背景スクリムタップ、②ヘッダーの `×` ボタン、③ Android ハードウェアバック / iOS スワイプダウン（`onRequestClose`）。**破棄する内容がないため確認ダイアログは出さない**（§12.1 の破棄確認とは別物）|

モーダル内に「決定」「クリア（全解除）」に相当するボタンはない。チップタップの都度 `onChange` が即座に呼ばれて選択状態が親（`PostComposer`）へ反映されるため、モーダルはいつ閉じても選択内容は保持される。

### 7.5 選択の挙動（常に任意・即時反映・最大3）

- ジャンル一覧: `GENRE_CATEGORY_ORDER`（松柏類 / 雑木類 / 草もの / 用品・道具 / 施設・イベント / その他）の 6 種。旧仕様から変更なし
- 上限: `MAX_GENRES_PER_POST`（3 件）。旧仕様から変更なし。3 件選択済みの状態で未選択チップをタップしても何も起こらない
- **0 件のまま投稿・保存できる。** ジャンルは常に任意であり、送信ボタンの活性条件（§10.4）にジャンル選択数は含まれない
- 「必須」を示すアスタリスク等の視覚マークは一切つけない。任意であることはプレースホルダー文言「ジャンルを選択（任意）」で伝える

### 7.6 accessibility（トリガー・モーダル・チップ）

| 要素 | 仕様 |
|------|------|
| トリガーボタン | §7.3 参照 |
| モーダル | `accessibilityViewIsModal` を付与し、背景要素へのフォーカスを防ぐ |
| 背景スクリム / 閉じるボタン | `accessibilityRole="button"` / `accessibilityLabel="閉じる"` |
| チップ | `accessibilityRole="checkbox"` / `accessibilityState: { checked, disabled: isExhausted }` / 非選択「{ジャンル名}を選択」・選択済み「{ジャンル名}の選択を解除」/ `accessibilityHint="3つまで選択できます。現在{N}つ選択中"`（選択数・選択有無に関わらず常時付与）|

### 7.7 旧仕様（撤回 — 実装・Web 本番と乖離していたため無効）

> 以下は 2026-06-19 版の元記述。**取り消し済みであり現在は無効。** §7.1〜§7.6 に置き換えられた。変更履歴の参照用に残す。

**（旧）7.1 チップレイアウト**

```
[松柏類]  [雑木類]  [草もの]  [用品・道具]  [施設・イベント]  [その他]
```

- 横並び `flexWrap: "wrap"` / `gap: spacing2`（8pt）。本文入力エリアの直下に**常時展開表示**（トリガーボタンやモーダルは存在しない設計だった）
- 各チップの高さ: 36pt（`hitSlop` で 44pt 確保）

**（旧）7.3 最大選択数制御**

- 3 件選択済みの状態で未選択チップをタップ → 何も起こらない（disabled）
- 3 件選択済みの状態をアクセシビリティで伝える: `accessibilityHint="3つまで選択できます。現在{N}つ選択中"`

**（旧）7.4 accessibilityLabel（各チップ）**

- 非選択: 「{ジャンル名}を選択」
- 選択済み: 「{ジャンル名}の選択を解除」
- `accessibilityRole: "checkbox"` / `accessibilityState: { checked: boolean }`

（旧 7.2「チップの状態」の内容は §7.2 として現行版にもほぼそのまま引き継いだため、ここでは重複記載しない。差分は §7.2 末尾の取り消し注記を参照）

**撤回理由:**

1. チップを本文直下に常時・無条件で展開表示し、見出しに「（最大3つまで選択）」という上限表記はあっても「（任意）」の表記が一切なかった。この見た目は事実上「選択して当然」の必須項目であるかのような誤解を与える仕様だった。
2. **Web 本番（`Bon_Log_cfw` の GenreSelector）はジャンル選択を常に任意としている。** プレースホルダーに明示的に「ジャンルを選択（任意）」と表示し、0 件のまま投稿できる。旧仕様はこの Web の挙動と乖離していた。
3. UI 方式自体も Web 本番の「トリガーボタン → ドロップダウン」と異なっていた。限られた画面幅で全チップを常時展開すると本文入力エリアを圧迫し、モバイルの縦スクロール体験としても最適ではなかった。
4. 現行実装 `components/post/GenreSelector.tsx` は上記 1〜3 を是正済み。本ドキュメントは §7.1〜§7.6 を現状の正とする。

## 8. 画像選択グリッド（ImageAttachmentGrid）の詳細仕様

### 8.1 グリッドレイアウト

- 横スクロール可能な行（`ScrollView` horizontal）
- 各サムネイル: 正方形 80 x 80pt / 角丸 `radiusMd`（8pt）
- サムネイル間隔: `spacing2`（8pt）

```
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│ +    │ │ 🗑 │ │ 🗑 │ │ 🗑 │ │ 🗑 │
│ 追加 │ │[画像]│ │[画像]│ │[画像]│ │[画像]│
└──────┘ └──────┘ └──────┘ └──────┘ └──────┘
              ↑ 各サムネイルの右上に削除ボタン（20pt 丸アイコン）
```

### 8.2 「+ 追加」ボタン

- 同じ 80 x 80pt の正方形セル
- 背景: `colorSurfaceMuted`（`#f0f0f0`）/ 角丸 `radiusMd` / ボーダー `colorBorderLight` 1pt dashed（ダッシュ）
- 中央に `+` アイコン（24pt）/ `colorTextSecondary`
- 上限に達した場合（無料: 4 枚、プレミアム: 6 枚）: 非表示にする（スペースを詰める）
- `accessibilityLabel`: 「画像を追加」
- タップ → `expo-image-picker` を起動する（ライブラリから選択。複数選択可能で残り枚数分まで選択できる）

### 8.3 サムネイルセル

- 画像表示: `expo-image` / `contentFit: "cover"` / `transition: durationFast`
- 削除ボタン: 右上に重なる丸ボタン（直径 24pt）
  - 背景: `colorActionPrimary`（`#2e2e2e`）/ `radiusFull`
  - アイコン: X（12pt）/ `colorActionPrimaryText`
  - タップターゲット: `hitSlop: { top: 10, right: 10, bottom: 10, left: 10 }` で 44pt 確保
  - `accessibilityLabel`: 「画像を削除」
- `accessibilityLabel`（セル全体）: 「添付画像{N}枚目」

### 8.4 並び順の変更（ドラッグ）

- ドラッグ操作で並び順を変更できる（`react-native-reanimated` + ドラッグ処理）
- ドラッグ中のサムネイルは 1.05 倍スケールアップする
- **frontend への申し送り:** ドラッグ並び替えの実装ライブラリ選定は frontend に委ねる（`react-native-draggable-flatlist` 等）。MVP では並び替え未実装でも受け入れ可とし、削除・再追加で対応できることを UI で示す

### 8.5 プレミアム上限（6 枚）との関係

- `isPremium=false` の場合、4 枚目追加後に「+ 追加」ボタンが消える
- `isPremium=true` の場合、6 枚目追加後に「+ 追加」ボタンが消える
- 編集時に既存画像が 5〜6 枚（プレミアム時に作成）の場合: `isPremium=false` なら画像を削除はできるが追加はできない（グレーのロックメッセージを表示）

## 10. ヘッダー（PostComposerHeader）の詳細仕様

### 10.1 ボタン配置

```
[キャンセル]       新規投稿 / 投稿を編集       [投稿する / 保存する]
```

- 高さ: 54pt（セーフエリア上端との間に `spacing4` を確保）
- 下ボーダー: 1pt `colorBorderLight`
- 背景: `colorSurfaceWashi`（`#f7f7f7`）
- タイトル: `textMd`（15pt）/ `fontWeight: 600` / `colorTextPrimary` / 中央揃え

### 10.2 左ボタン（キャンセル）

- テキストボタン / `textMd` / `colorTextSecondary` / タップターゲット 44pt 以上
- タップ → §12.1 の破棄確認フローへ

### 10.3 右ボタン（送信 / 保存）

| 状態 | 見た目 | 条件 |
|------|-------|------|
| 通常（活性）| `colorActionPrimary` テキスト / `textMd` / `fontWeight: 600` | 送信条件を満たしている |
| 無効（非活性）| `colorTextTertiary` テキスト（不透明度 0.4）| 送信条件未達 |
| 送信中 | テキスト非表示 + スピナー（`colorActionPrimary` / サイズ 18pt）| API 呼び出し中 |

- `accessibilityLabel`: 「投稿する」または「変更を保存する」
- `accessibilityState: { disabled: boolean }`

### 10.4 送信ボタンの活性条件

以下を**すべて**満たした場合のみ活性化する（実装: `components/post/PostComposer.tsx` の `canSubmit = hasContent && !isContentOverLimit && !isSubmitting && isPollValid`）:

1. 本文が 1 文字以上 OR 画像が 1 枚以上添付されている（`hasContent`）
2. 本文文字数が上限以内（無料: 500 字、プレミアム: 2000 字）
3. 送信中でない
4. アンケート（`PollAttachment`）を添付している場合、選択肢が 2 つ以上かつすべて入力されている（`isPollValid`。アンケートを添付していない場合はこの条件を常に満たす）

ジャンル選択数（§7）はこの条件に含まれない（0 件でも送信可）。

**（2026-07-13 追記）:** 条件 4 は実装済みの `PollAttachment`（アンケート添付。新規投稿のみ）に対応するチェックだが、本書はこれまで §10.4 に記載していなかった。§7 のジャンル是正に付随して発覚したため、条件のみ本改訂で追記した。`PollAttachment` 自体のワイヤーフレーム・コピー案等の完全な仕様化は本書の対象外（§16 参照）。

## 12. エッジケース

### 12.1 破棄確認（未入力のまま / 入力中に「キャンセル」または戻る操作）

入力済みコンテンツがある場合（本文テキスト / ジャンル選択 / 画像添付のいずれか）、破棄確認を出す。
「何も変更していない場合」は確認なしで即閉じる。

**iOS / Android 差異:**
- iOS: `Alert.alert()` のネイティブダイアログ
- Android: カスタムモーダルダイアログ

ダイアログの内容:

```
タイトル: 「投稿を破棄しますか？」
本文:     「入力した内容は保存されません。」
ボタン:
  [入力を続ける]  — テキストボタン / colorTextSecondary
  [破棄する]      — 塗りつぶし / colorError / colorTextInverse
```

**iOS のスワイプダウン:** モーダル中段以上からのスワイプダウンで閉じようとした場合も同じ確認を出す（`onRequestClose` に組み込む）。

### 12.2 空状態

- 本文・画像ともに未入力の場合: 「投稿する」ボタンが disabled のため操作不可
- 空状態の画面をエラーとして扱わない

### 12.3 エラー（送信失敗）

| エラー種別 | 表示箇所 | エラー定数カテゴリ |
|-----------|---------|-----------------|
| 文字数超過 (400) | `ComposerFormError` | `ERR_POST_CONTENT_TOO_LONG` 相当 |
| 投稿数上限 (429) | `ComposerFormError` | `ERR_POST_DAILY_LIMIT` 相当 |
| 画像アップロード失敗 | `ComposerFormError` | `ERR_IMAGE_UPLOAD_FAILED` 相当 |
| 権限なし (403) | `ComposerFormError` | `ERR_FORBIDDEN` 相当 |
| ネットワークエラー | `ComposerFormError` | `ERR_NETWORK` 相当 |
| サーバーエラー (5xx) | `ComposerFormError` | `ERR_SERVER` 相当 |

定数の正確な名称は `lib/constants/errors.ts` に frontend が定義する際に確定させること。

### 12.4 オフライン

- 送信ボタンタップ時にオフラインを検知した場合: API 呼び出しを行わず、`ComposerFormError` にオフラインエラー文言を表示する（`ERR_OFFLINE_ACTION` 相当）
- `OfflineBanner` を画面上部に表示する
- オフライン中でも入力を継続できる（送信だけブロック）

### 12.5 編集時のエラー

- 編集対象投稿の取得失敗（404）: `ScreenError`（title: 「投稿が見つかりません」/ `ERR_POST_NOT_FOUND`）を表示する
- ネットワークエラーでの初期値取得失敗: `ScreenError`（onRetry: 再フェッチ）

### 12.6 画像選択のエラー

- 写真へのアクセス権限がない場合: `Alert.alert()` で「写真へのアクセスが許可されていません。設定アプリから許可してください。」と表示し、iOS では設定アプリへの誘導ボタンを出す（`Linking.openSettings()`）
- Android での権限リクエストは `expo-image-picker` が自動で行う

| 文字数超過時カウンタ補足 | 「{N}文字オーバーしています。削除してから投稿してください。」（フォームエラーバナー）|

**（2026-07-12 取り消し注記）:** 旧「ジャンルセクション見出し」行＝「ジャンル（最大3つまで）」は撤回済み。実装では見出しは「ジャンル」のみで、上限・任意である旨はトリガーのプレースホルダーとカウンタで表現する（詳細は §7.7）。

## 14. アクセシビリティ

| 要素 | 仕様 |
|------|------|
| 本文テキストエリア | `accessibilityLabel="投稿内容を入力"` / `accessibilityHint` で文字数上限を伝える |
| 文字数カウンタ | `accessibilityLabel="{N}文字 / {上限}文字"` / 上限超過時は `accessibilityRole="alert"` を付与 |
| ジャンルトリガーボタン | `accessibilityRole="button"` / `accessibilityLabel` は表示中テキスト（プレースホルダー、または選択中ジャンル名の連結）と同一 / `accessibilityHint="タップしてジャンルを選択します"` / `accessibilityState: { disabled }`（詳細 §7.3）|
| ジャンルチップ（選択モーダル内）| `accessibilityRole="checkbox"` / `accessibilityState: { checked, disabled }` / `accessibilityLabel` で選択状態を伝える（詳細 §7.6）|
| 「+ 追加」ボタン（画像）| `accessibilityRole="button"` / `accessibilityLabel="画像を追加"` |
| 削除ボタン（画像）| `accessibilityRole="button"` / `accessibilityLabel="この画像を削除"` |
| 送信ボタン | `accessibilityRole="button"` / `accessibilityState: { disabled }` |
| フォームエラーバナー | `accessibilityRole="alert"` / `accessibilityLiveRegion="assertive"` |
| モーダル全体 | `accessibilityViewIsModal={true}` でバックグラウンドへのフォーカスを防ぐ |
