## 1. 概要・目的

盆栽の記録をテキスト・画像・ジャンルタグとともに投稿する画面。
新規投稿と既存投稿の編集の 2 パターンを 1 仕様で定義する（レイアウトは共通）。

- 新規投稿: FAB タップ → モーダルで表示（`posts/new`）
- 投稿編集: 投稿詳細の「⋮ メニュー → 編集」→ モーダルで表示（`posts/[id]/edit`）

### 1.1 解決する問題

- モバイルでの投稿作成導線を確立する（Web の `/posts/new` に相当する機能のモバイル版）
- 無料ユーザーとプレミアムユーザーの制約を画面上で明示し、アップグレードへの自然な導線とする

## 5. データの流れ

### 5.1 新規投稿

1. ユーザーが本文・ジャンル（個別ジャンルの id を最大3件・任意）・盆栽紐付け（自分の盆栽1件・任意）・画像を入力する
2. 「投稿する」タップ時、画像がある場合: presigned URL 取得 → R2 へ直接 PUT アップロード → URLをリクエストに含める
3. `POST /api/v1/posts`（実装 `lib/queries/posts.ts` の `useCreatePostMutation` で確認済み。body: `content` / `genreIds: string[]` / `mediaUrls` / `mediaTypes` / `poll`（任意）/ `bonsaiId: string | null`（任意。§7.9 参照））
4. 成功後: `posts.feed` / 自分の `users.detail` を `invalidateQueries`

### 5.2 投稿編集

1. 画面マウント時に既存投稿データ（`usePostQuery(id)`）を初期値としてフォームに展開する。ジャンルは `post.genres`（`{ id, name }[]`）から `genreIds: string[]` を導出して事前選択する（§7.5 参照）。**盆栽紐付け（`BonsaiSelector`）は編集画面に表示しない**ため、既存投稿の盆栽紐付けは編集画面上で確認・変更できない（§7.9.1 参照）
2. 変更があった項目のみサーバーへ送る（フロントエンドの実装に委ねる）
3. `PATCH /api/v1/posts/{id}`（想定エンドポイント。正本は OpenAPI）
4. 成功後: `posts.detail(id)` / `posts.feed` を `invalidateQueries`

**core に要確認:** PATCH エンドポイントが部分更新（差分のみ送信）か全件置き換えかによってリクエスト設計が変わる。**盆栽紐付け（`bonsaiId`）に限っては、実装 `lib/queries/posts.ts`（`UpdatePostParams.bonsaiId`）が「キー省略＝現状維持」の部分更新契約で設計されていることを確認済み**（§7.9.5 参照）。ただし `content` / `genreIds` / `mediaUrls` / `mediaTypes` は `UpdatePostParams` 上で必須（非 optional）フィールドのため、これらが差分のみ送信か全件送信かは本改訂の調査範囲外のまま未確認とする。

### 5.3 画像アップロード

- `expo-image-picker` で端末内の画像を選択する
- アップロード前に `expo-image-manipulator` で圧縮する（品質・リサイズの具体的な設定値は `lib/constants/` の定数。**frontend への申し送り**）
- presigned URL 取得 → R2 に直接 PUT する（`api-client.md` 例外規定）

### 5.4 必要なデータ項目（編集初期値）

| 項目 | 意味 |
|------|------|
| `content` | 本文テキスト |
| `genres` | ジャンル配列（`{ id, name }[]`）|
| `media` | 既存メディア配列（`{ id, url, type, sortOrder }[]`）|

**（2026-07-14 追記）:** `genres` は編集画面側で `post.genres.map(g => g.id)` として `genreIds: string[]`（`GenreSelector` の `selectedGenres` 初期値）に変換してから渡す（実装 `app/posts/[id]/edit/index.tsx`）。**盆栽紐付け（`bonsaiId`）は編集初期値に含まれない** — `BonsaiSelector` 自体を編集画面で描画しないため（§7.9.1）。

## 6. 本文入力エリア（PostBodyInput）の詳細仕様

### 6.1 テキストエリア

- `TextInput` の `multiline: true` / `scrollEnabled: true`（テキストが長くなった場合にエリア内でスクロール）
- 最小高さ: 120pt（3〜4 行分が最初から見える）
- 最大高さ: 制限なし（エリアは固定高さ。コンテナ全体が `ScrollView` でスクロール可能）
- `placeholder`: 「盆栽の記録をシェアしましょう...」
- `placeholderTextColor`: `colorTextTertiary`（`#8a8a8a`）
- フォント: `textBase`（14pt / lineHeight 22pt）/ `colorTextPrimary`
- 左右内側パディング: `spacing3`（12pt）/ 上下: `spacing3`（12pt）
- ボーダー: なし（テキストエリアはボーダーなしのフラットなデザイン。コンテナのセパレータで分割を示す）
- `keyboardType: "default"` / `autoCapitalize: "sentences"`（日本語入力に配慮）
- `returnKeyType: "default"`（改行を許容する）
- `accessibilityLabel`: 「投稿内容を入力」
- `accessibilityHint`: `isPremium` に応じた「最大500文字で入力してください」または「最大2000文字で入力してください」

### 6.2 文字数カウンタ

- 位置: テキストエリア右下（`textSm` / 12pt）
- 通常: `{現在文字数} / {上限}`（例: `123 / 500`）/ `colorTextSecondary`
- 残り 50 文字以下: `colorWarning`（`#b8860b`）に変化
- 上限超過（0 文字以下）: `colorError`（`#c21721`）に変化
- 上限超過の場合、「投稿する」ボタンを disabled にする

**（2026-07-13 第2回改訂）:** `colorError` の値表記を `#c0392b` から `#c21721` に修正した。根拠は `design-tokens.md` §11 を参照。

### 6.3 上限超過時のエラー表示

- 文字数カウンタを `colorError` に変化させるのみ（インラインエラーテキストは出さない）
- 「投稿する」ボタンが disabled になるため、ユーザーはカウンタを見て削減する

## 7. ジャンル選択（GenreSelector）・盆栽紐付け（BonsaiSelector）の詳細仕様

> **改訂注記（2026-07-12）:** 本セクションは実装 `components/post/GenreSelector.tsx` の確認結果に基づき全面改訂した。
>
> - **旧仕様（本文入力エリアの直下に全ジャンルチップを常時展開表示。見出しに「（最大3つまで選択）」という上限表記はあったが「（任意）」の表記が一切なく、実質必須の項目であるかのような見た目だった）は誤りだった。** 全文は §7.7 に取り消し注記として保存する。
> - **Web 本番（`Bon_Log_cfw` の `components/post/GenreSelector.tsx`）ではジャンル選択は常に任意である。** トリガーボタンのプレースホルダーに明示的に「ジャンルを選択（任意）」と表示し、1 つも選ばずに投稿できる（`maxSelections` 到達時のみ追加選択を disabled にする）。現行の Native 実装はこの Web 挙動に合わせて是正済みであり、本セクション（§7.1〜§7.6）はその実装を正として書き直したもの。
> - なお §10.4（送信ボタンの活性条件）は本改訂の前後を通じてジャンル選択数を条件に含めていない（本文 1 文字以上 OR 画像 1 枚以上のみが条件）。つまり送信そのものを妨げる必須項目ではなかったが、UI の見た目が実質必須であるかのような誤解を招く仕様だった、という点が今回の是正の要点である。

### 7.2 チップの状態（個別ジャンルチップ）

| 状態 | 背景 | テキスト | ボーダー |
|------|------|---------|---------|
| 非選択 | `colorActionSecondary`（`#e9e9e9`）| `colorActionSecondaryText`（`#2e2e2e`）| 1pt `colorBorderLight` |
| 選択済み | `colorActionPrimary`（`#2e2e2e`）| `colorActionPrimaryText`（`#ffffff`）| なし |

### 7.4 モーダルの構成・開閉・4状態（ローディング / エラー / 空 / 通常）

トリガーボタンをタップすると、画面下からスライドアップするモーダルシートで「カテゴリ見出し＋個別ジャンルチップ」の一覧を表示する（`Modal` / `animationType="slide"` / `transparent` / `accessibilityViewIsModal`）。一覧データは `useGenresQuery('post')`（`lib/queries/shops.ts`。ゲスト可・マスタ系データとして長めの `staleTime`（`STALE_TIME_MASTER`）で取得）から取得し、`groupGenresByCategory`（`lib/utils/group-genres-by-category.ts`）でカテゴリ単位にグルーピングしてから描画する。

```
┌─────────────────────────────────────────────────────────────┐
│ ░░░░░░░░░░░░░░░░ 背景スクリム（タップで閉じる） ░░░░░░░░░░░░ │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ ジャンルを選択                                    [×]   │ │ ← シートヘッダー
│ ├─────────────────────────────────────────────────────────┤ │
│ │ 松柏類                                                  │ │ ← カテゴリ見出し（textXs）
│ │ [黒松] [五葉松] [真柏]                                   │ │ ← 個別ジャンルチップ（flexWrap）
│ │                                                         │ │
│ │ 雑木類                                                  │ │ ← 次のカテゴリ見出し
│ │ [紅葉] [欅] [楓]                                        │ │
│ │                                                         │ │
│ │ ...（用品・道具 / 施設・イベント / その他 と続く）        │ │
│ │                                                         │ │
│ │ 2/3 選択中                                              │ │ ← 1件以上選択時のみ（モーダル内にも表示）
│ └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

| 要素 | 仕様 |
|------|------|
| 背景スクリム | `colorScrimLight`（`rgba(0, 0, 0, 0.40)`）/ タップでモーダルを閉じる |
| シート | 背景 `colorBackground` / 上端 2 隅のみ `radiusMd`（8pt）/ `shadowWashiLg` / 最大高さは画面の 70%（`maxHeight: "70%"`）/ 下端にセーフエリア分の余白（`insets.bottom`）を確保 |
| シートヘッダー | 背景 `colorSurfaceWashi`（`#f7f7f7`）/ 下ボーダー 1pt `colorBorderLight` / タイトル「ジャンルを選択」（`textLg` / `colorTextPrimary`）/ 右端に閉じるボタン（`close` アイコン 22pt・タップターゲット 44×44pt）|
| カテゴリ見出し | `textXs` / `colorTextTertiary` / `letterSpacingTight`。タップ不可の静的テキスト（`accessibilityRole` の指定なし）。該当ジャンルが1件もないカテゴリは見出し自体を生成しない |
| チップグリッド（カテゴリ内） | `flexWrap: "wrap"` / `gap: spacing2`（8pt）|
| スクロールコンテナ全体 | カテゴリ間の縦間隔含め `gap: spacing4`（16pt）/ 内側パディング `spacing4`（16pt。下端のみ `spacing3`）|
| 閉じる手段 | ①背景スクリムタップ、②ヘッダーの `×` ボタン、③ Android ハードウェアバック / iOS スワイプダウン（`onRequestClose`）。**破棄する内容がないため確認ダイアログは出さない**（§12.1 の破棄確認とは別物）|

モーダル内に「決定」「クリア（全解除）」に相当するボタンはない。チップタップの都度 `onChange` が即座に呼ばれて選択状態が親（`PostComposer`）へ反映されるため、モーダルはいつ閉じても選択内容は保持される。

**カテゴリの表示順（重要）:** サーバー API（`GET /api/v1/genres?type=post`）は `items` をカテゴリのアルファベット順で返し、`GENRE_CATEGORY_ORDER`（松柏類 / 雑木類 / 草もの / 用品・道具 / 施設・イベント / その他）の意図した表示順とは一致しない。クライアントは `groupGenresByCategory` を介してこの順序を `GENRE_CATEGORY_ORDER` に整列し直してから描画する（Web 版はサーバー側で組み立てた `Record<string, Genre[]>` を受け取る点は異なるが、最終的な「カテゴリ見出し→個別ジャンルチップ」という描画構造は同じ）。`GENRE_CATEGORY_ORDER` に存在しないカテゴリ（サーバー側で将来カテゴリが追加された場合）は、既知カテゴリの後ろに出現順のまま追加表示する安全側のフォールバックになっている。

**4状態（データ取得: `useGenresQuery('post')`）:**

| 状態 | 条件 | 表示 |
|------|------|------|
| ローディング | `isLoading === true` | シートスクロールエリア中央にスピナーのみ（`ActivityIndicator` / `colorActionPrimary`）。`accessibilityRole="progressbar"` / `accessibilityLabel="ジャンルを読み込み中"` |
| エラー | `isLoading === false && isError === true` | `ERR_LOAD_FAILED`（「読み込みに失敗しました。」）を `colorError` テキストで表示 + 「再試行」ボタン（`refetch()` を呼ぶ）。ブロック全体に `accessibilityRole="alert"` / `accessibilityLiveRegion="assertive"` |
| 空 | `isLoading === false && isError === false && categoryGroups.length === 0` | 「選択できるジャンルがありません」（`colorTextTertiary`）。サーバーがジャンルマスタを0件返した場合のみ発生する稀なケース |
| 通常 | 上記いずれでもない | カテゴリ見出し＋個別ジャンルチップの一覧を表示 |

これら4状態はモーダルを開いた後（シート内部）でのみ発生する。トリガーボタン自体は読み込み中・エラー中でも常に押せる状態で表示され（プレースホルダー「ジャンルを選択（任意）」のまま）、状態の違いはモーダルを開いて初めて可視化される。

### 7.5 選択の挙動（常に任意・即時反映・最大3・カテゴリ横断）

- 選択可能な項目は `GENRE_CATEGORY_ORDER` の6カテゴリの下に紐づく**個別ジャンル**（`useGenresQuery('post')` が返す実データ。カテゴリごとの件数はサーバーのジャンルマスタ次第であり、6件固定ではない）。6カテゴリ自体は選択対象ではなく、あくまでチップをグルーピングする見出しである
- 上限: `MAX_GENRES_PER_POST`（3 件）。**カテゴリをまたいで合計3件**まで選択できる（例: 松柏類から2件＋雑木類から1件、のような組み合わせも可）。3 件選択済みの状態で未選択チップをタップしても何も起こらない
- **0 件のまま投稿・保存できる。** ジャンルは常に任意であり、送信ボタンの活性条件（§10.4）にジャンル選択数は含まれない
- 「必須」を示すアスタリスク等の視覚マークは一切つけない。任意であることはプレースホルダー文言「ジャンルを選択（任意）」で伝える
- 送信・保持する値は選択した**個別ジャンルの実 `id`（cuid）の配列**。カテゴリ名の文字列を値として扱うことはない（§4.1 の props 定義参照）
- 編集時は既存投稿の `genres`（`{ id, name }[]`）から `id` を取り出した配列で事前選択する（§5.4 参照）

**（2026-07-14 是正）:** 旧稿は「ジャンル一覧: `GENRE_CATEGORY_ORDER` の6種。旧仕様から変更なし」としており、6カテゴリそのものが選択対象であるかのように記述していた。実際の選択対象は6カテゴリの下に紐づく個別ジャンル（件数はサーバーデータ依存）である。上限3件・常に任意・0件可という制約の性質自体に変更はない。

### 7.9 盆栽紐付け（BonsaiSelector）の詳細仕様

投稿に自分の盆栽1件を紐付ける、**新規投稿時のみ**の任意項目。Web 版 `BonsaiSelectorSection`（`<select>` ドロップダウン）に相当し、実装は `components/post/BonsaiSelector.tsx`。§7.1〜§7.8 の `GenreSelector` とレイアウトの骨格（トリガーボタン→モーダル）を流用しつつ、モーダル内は「チェックマーク付き単一選択リスト」（都道府県選択で使われている `components/shops/PrefecturePickerModal.tsx` 等と同系統のパターン）である点が `GenreSelector`（複数選択チップ）と異なる。

#### 7.9.1 適用範囲（新規投稿限定・Web パリティ）

| 項目 | 扱い |
|------|------|
| 新規投稿（`posts/new`） | 表示する（本文入力エリア・ジャンル選択の下、画像エリアの上に配置。§2.1 参照）|
| 投稿編集（`posts/[id]/edit`） | **表示しない。** Web 版 `PostEditForm` にも対応する UI が存在せず、`bonsaiId` を編集で変更する手段がないため（Web パリティ）。編集では常に既存の紐付けを維持したままサーバーへ送信する（三値契約の「現状維持」。§7.9.5 参照）|

#### 7.9.2 全体構成（トリガーボタン → モーダル）

```
未選択時（デフォルト）:
┌─────────────────────────────────────────────────────────────┐
│ 関連する盆栽（任意）                                          │
```

- ラベル「関連する盆栽（任意）」: `textXs` / `colorTextTertiary` / `letterSpacingTight`（`GenreSelector` の「ジャンル」ラベルと同じスタイル）

#### 7.9.3 モーダルの構成と状態（ローディング / エラー / 通常）

トリガーをタップすると、`GenreSelector` と同じ骨格（背景スクリム + 下からスライドアップするシート + `shadowWashiLg`）のモーダルを開く。シートヘッダーのタイトルは「関連する盆栽を選択」。一覧データは `useBonsaiListQuery()`（`lib/queries/bonsai.ts`。無限スクロール・自分の盆栽のみが対象・`BONSAI_PAGE_SIZE` 件ずつ取得）。

```
┌─────────────────────────────────────────────────────────────┐
│ ░░░░░░░░░░░░░░░░ 背景スクリム（タップで閉じる） ░░░░░░░░░░░░ │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ 関連する盆栽を選択                                [×]   │ │ ← シートヘッダー
│ ├─────────────────────────────────────────────────────────┤ │
│ │ 選択しない                                          ✓   │ │ ← 先頭に固定（常に存在する行）
│ │ ──────────────────────────────────────────────────── │ │
│ │ 黒松の一鉢 (黒松)                                        │ │ ← 自分の盆栽一覧（FlatList）
│ │ ──────────────────────────────────────────────────── │ │
│ │ 五葉松の盆栽                                            │ │
│ │ ...（無限スクロール。末尾でフッタースピナー）             │ │
│ └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

| 要素 | 仕様 |
|------|------|
| 行リスト | `FlatList`。先頭に固定の「選択しない」行（値 `null`。Web の `<option value="">選択しない</option>` に相当）+ 自分の盆栽一覧。行区切りは 1pt `colorBorderLight` |
| 行の高さ | 最小 48pt |
| 行の選択状態 | 選択中の行のみ右端に `checkmark` アイコン（18pt / `colorActionPrimary`）を表示し、テキストを `colorActionPrimary` / `fontWeight: 600` にする |
| 行タップ | 即座に `onChange` を呼び、モーダルを閉じる（`GenreSelector` のチップと異なり、単一選択のため選択と同時にクローズする）|
| 無限スクロール | リスト末尾で `fetchNextPage`。取得中はリスト末尾にスピナー（`colorActionPrimary`）|
| accessibilityRole（各行） | `"radio"` / `accessibilityState: { selected }` / `accessibilityLabel` は表示ラベルと同一 |

**状態:**

| 状態 | 条件 | 表示 |
|------|------|------|
| ローディング | `isLoading === true` | シート内中央にスピナーのみ。`accessibilityRole="progressbar"` / `accessibilityLabel="盆栽を読み込み中"` |
| エラー | `isLoading === false && isError === true` | `ERR_LOAD_FAILED`（「読み込みに失敗しました。」）+「再試行」ボタン（`refetch()`）。`accessibilityRole="alert"` / `accessibilityLiveRegion="assertive"` |
| 通常 | 上記いずれでもない | 「選択しない」+ 自分の盆栽一覧の `FlatList`（「選択しない」行は常に存在するため、モーダル内でリストが空になることはない）|

自分の盆栽が0件の場合はモーダルを開く前の段階でセクションごと非表示になる（§7.9.4 参照）ため、モーダル内に独立した「空」状態は存在しない。

#### 7.9.4 盆栽 0 件時は非表示（Web パリティ）

Web 版 `BonsaiSelectorSection` は `bonsaiList.length === 0` の場合 `return null` する。Native も同じ挙動を再現する: 「読み込みが完了し（`isLoading === false`）、エラーでもなく（`isError === false`）、盆栽が1件もなく（`allItems.length === 0`）、次ページも存在しない（`hasNextPage !== true`）」の**すべてを満たしたときのみ**、`BonsaiSelector` はセクション全体（ラベル・トリガーとも）を描画しない。

- 初回ロード中・エラー中は「0件と確定していない」ため、この間はセクションを通常どおり表示する（トリガーを開くとローディング/エラー状態が見える）。0件が確定した時点でセクションごと消える
- **既知のエッジケース:** モーダルを開いた状態のまま０件が確定した場合（初回ロード中に開いて、結果が0件だったケース）、次の再描画で `BonsaiSelector` がセクションごと非表示になるため、モーダルも同時に閉じる（明示的なクローズアニメーションを伴わない）。発生頻度は低いが実装上の挙動として記録する
- 送信可否には影響しない。ジャンル同様、盆栽紐付けの有無は送信ボタンの活性条件（§10.4）に含まれない

#### 7.9.5 データの送受信（三値契約・404エラー）

- 送信する値は `bonsaiId: string | null`（選択中の盆栽の id、または「選択しない」の場合は `null`）
- 作成ミューテーション（`useCreatePostMutation` の `CreatePostParams.bonsaiId?: string | null`）にそのまま渡す
- 編集では `BonsaiSelector` 自体を描画しないため `selectedBonsaiId` の概念がなく、更新ミューテーション（`UpdatePostParams.bonsaiId?: string | null`）へは**キー自体を渡さない**。サーバーの PATCH は部分更新契約のため、キー省略＝「現状維持」・`null` 明示＝「紐付け解除」・文字列＝「紐付け設定」という三値制御になるが、編集フォームからは解除・変更の経路自体が存在しない（本 UI の対象外）
- 他人が所有する盆栽 ID、または存在しない盆栽 ID を指定した場合、サーバーは 404 NOT_FOUND を返す（所有者以外への存在有無の秘匿のため、他人所有と不存在を区別しない）。この場合のユーザー向け文言・実装は §12.3 を参照

- ドラッグ操作で並び順を変更できる（`react-native-reanimated` + ドラッグ処理）

### 8.5 プレミアム上限（6 枚）との関係

- `isPremium=true` の場合、6 枚目追加後に「+ 追加」ボタンが消える

1. 本文が 1 文字以上 OR 画像が 1 枚以上添付されている（`hasContent`）
2. 本文文字数が上限以内（無料: 500 字、プレミアム: 2000 字）
3. 送信中でない
4. アンケート（`PollAttachment`）を添付している場合、選択肢が 2 つ以上かつすべて入力されている（`isPollValid`。アンケートを添付していない場合はこの条件を常に満たす）

**（2026-07-13 追記）:** 条件 4 は実装済みの `PollAttachment`（アンケート添付。新規投稿のみ）に対応するチェックだが、本書はこれまで §10.4 に記載していなかった。§7 のジャンル是正に付随して発覚したため、条件のみ本改訂で追記した。`PollAttachment` 自体のワイヤーフレーム・コピー案等の完全な仕様化は本書の対象外（§16 参照）。

## 12. エッジケース

### 12.1 破棄確認（未入力のまま / 入力中に「キャンセル」または戻る操作）

入力済みコンテンツがある場合（本文テキスト / ジャンル選択 / 画像添付のいずれか）、破棄確認を出す。
「何も変更していない場合」は確認なしで即閉じる。

**iOS / Android 差異:**
- iOS: `Alert.alert()` のネイティブダイアログ
- Android: カスタムモーダルダイアログ

ダイアログの内容:

```
タイトル: 「投稿を破棄しますか？」
本文:     「入力した内容は保存されません。」
ボタン:
  [入力を続ける]  — テキストボタン / colorTextSecondary
  [破棄する]      — 塗りつぶし / colorError / colorTextInverse
```

**iOS のスワイプダウン:** モーダル中段以上からのスワイプダウンで閉じようとした場合も同じ確認を出す（`onRequestClose` に組み込む）。

### 12.2 空状態

- 本文・画像ともに未入力の場合: 「投稿する」ボタンが disabled のため操作不可
- 空状態の画面をエラーとして扱わない

### 12.3 エラー（送信失敗）

| エラー種別 | 表示箇所 | エラー定数カテゴリ |
|-----------|---------|-----------------|
| 文字数超過 (400) | `ComposerFormError` | `ERR_POST_CONTENT_TOO_LONG` 相当 |
| 投稿数上限 (429) | `ComposerFormError` | `ERR_POST_DAILY_LIMIT` 相当 |
| 画像アップロード失敗 | `ComposerFormError` | `ERR_IMAGE_UPLOAD_FAILED` 相当 |
| 権限なし (403) | `ComposerFormError` | `ERR_FORBIDDEN` 相当 |
| ネットワークエラー | `ComposerFormError` | `ERR_NETWORK` 相当 |
| サーバーエラー (5xx) | `ComposerFormError` | `ERR_SERVER` 相当 |
| 指定した盆栽が見つからない (404。**新規投稿で `bonsaiId` を指定した場合のみ**) | `ComposerFormError` | `ERR_BONSAI_NOT_FOUND`（`messageForPostBonsaiError` 経由）— 文言「指定した盆栽が見つかりません。」|

定数の正確な名称は `lib/constants/errors.ts` に frontend が定義する際に確定させること。**（2026-07-14 追記）盆栽 404 の行のみ実装 `components/post/PostComposer.tsx`（`resolveCreateError`）・`lib/constants/errors.ts`（`ERR_BONSAI_NOT_FOUND` / `messageForPostBonsaiError`）で検証済み。** 他人所有・不存在のいずれの盆栽 ID でも同一の 404 NOT_FOUND として返るため区別できない（存在有無の秘匿設計。§7.9.5 参照）。このエラー分岐は投稿編集（`resolveUpdateError`）には存在しない — 編集フォームは `BonsaiSelector` を描画せず `bonsaiId` を送信しないため（§7.9.1）。

### 12.4 オフライン

- 送信ボタンタップ時にオフラインを検知した場合: API 呼び出しを行わず、`ComposerFormError` にオフラインエラー文言を表示する（`ERR_OFFLINE_ACTION` 相当）
- `OfflineBanner` を画面上部に表示する
- オフライン中でも入力を継続できる（送信だけブロック）

### 12.5 編集時のエラー

- 編集対象投稿の取得失敗（404）: `ScreenError`（title: 「投稿が見つかりません」/ `ERR_POST_NOT_FOUND`）を表示する
- ネットワークエラーでの初期値取得失敗: `ScreenError`（onRetry: 再フェッチ）

### 12.6 画像選択のエラー

- 写真へのアクセス権限がない場合: `Alert.alert()` で「写真へのアクセスが許可されていません。設定アプリから許可してください。」と表示し、iOS では設定アプリへの誘導ボタンを出す（`Linking.openSettings()`）
- Android での権限リクエストは `expo-image-picker` が自動で行う

## 14. アクセシビリティ

| 要素 | 仕様 |
|------|------|
| 本文テキストエリア | `accessibilityLabel="投稿内容を入力"` / `accessibilityHint` で文字数上限を伝える |
| 文字数カウンタ | `accessibilityLabel="{N}文字 / {上限}文字"` / 上限超過時は `accessibilityRole="alert"` を付与 |
| ジャンルトリガーボタン | `accessibilityRole="button"` / `accessibilityLabel` は表示中テキスト（プレースホルダー、または選択中の個別ジャンル名の連結）と同一 / `accessibilityHint="タップしてジャンルを選択します"` / `accessibilityState: { disabled }`（詳細 §7.3）|
| ジャンルチップ（選択モーダル内）| `accessibilityRole="checkbox"` / `accessibilityState: { checked, disabled }` / `accessibilityLabel` で選択状態を伝える（詳細 §7.6）|
| 盆栽紐付けトリガーボタン | `accessibilityRole="button"` / `accessibilityLabel` は表示中テキスト（「選択しない」、または選択中の盆栽名）と同一 / `accessibilityHint="タップして関連する盆栽を選択します"` / `accessibilityState: { disabled }`（詳細 §7.9.2）|
| 盆栽選択モーダルの行 | `accessibilityRole="radio"` / `accessibilityState: { selected }` / `accessibilityLabel` は行の表示ラベルと同一（詳細 §7.9.3）|
| 「+ 追加」ボタン（画像）| `accessibilityRole="button"` / `accessibilityLabel="画像を追加"` |
| 削除ボタン（画像）| `accessibilityRole="button"` / `accessibilityLabel="この画像を削除"` |
| 送信ボタン | `accessibilityRole="button"` / `accessibilityState: { disabled }` |
| フォームエラーバナー | `accessibilityRole="alert"` / `accessibilityLiveRegion="assertive"` |
| モーダル全体 | `accessibilityViewIsModal={true}` でバックグラウンドへのフォーカスを防ぐ |
