# アカウント削除フロー UI/UX 仕様 — Bon_Log Native

作成日: 2026-06-19
対象画面:
- `settings/account` — アカウント設定（削除導線を含む）

準拠規則:
- `.claude/rules/store-compliance.md`（Google Play アカウント削除要件 — MVP 必須）
- `auth-tokens.md`（ログアウト手順）

前提:
- `design-tokens.md` §2.3 のトークン名を使用する
- `navigation-structure.md` §4.3（アカウント設定画面）・§5.1（スタックヘッダー）に準拠

- `auth-forms.md` §0.3（FormErrorMessage）を流用する

## 1. 概要・目的

アカウントを削除したいユーザーがアプリ内から確実に削除操作を実行できる導線を提供する。

**これは Google Play 審査の必須要件である。** `store-compliance.md` の要件「アカウント作成があるアプリはアプリ内からのアカウント削除導線が必須」に対応する。Web にアカウント削除機能が存在しているとしても、アプリ内から実行できなければポリシー違反となる。

### 1.1 解決する問題

- アプリ内にアカウント削除導線がない状態を解消する
- 不可逆操作であることをユーザーが確実に理解したうえで実行できる UI フローを設計する
- プレミアム購読中ユーザーへの適切な案内（Google Play の購読管理について）を行う

### 2.2 「危険ゾーン」セクションの設計

- セクション見出し「危険ゾーン」: `textSm`（12pt）/ `colorError`（`#c0392b`）/ `letterSpacingWidest`（1.5）/ 上下に `spacing2` のセパレータ
- 「アカウントを削除する」リスト行:
  - 高さ: 56pt（タップターゲット確保）
  - テキスト: `textMd`（15pt）/ `colorError`（`#c0392b`）
  - 左端にアイコン: AlertTriangle 系 / サイズ 20pt / `colorError`
  - 背景: `colorBackground`（通常と同じ白背景。赤背景は使わない — ユーザーが誤タップを避けるため視認性は保ちつつ過度な強調は避ける）
  - 右端にシェブロン（›）: 20pt / `colorTextTertiary`
  - 押下時フィードバック: 背景が `colorErrorBg`（`#fdf0ef`）に変化（`durationFast: 200ms`）
  - `accessibilityRole="button"` / `accessibilityLabel="アカウントを削除する"` / `accessibilityHint="この操作は取り消せません。タップすると確認画面が表示されます。"`
- ヒントテキスト: 「アカウントを削除すると、投稿・コメント・フォロー関係を含むすべてのデータが削除され、元に戻すことはできません。」
  - `textSm`（12pt）/ `colorTextSecondary` / `spacing3`（12pt）上マージン

## 3. 削除フロー全体

```
settings/account で「アカウントを削除する」をタップ
  → §4 の第1ダイアログ（警告）を表示
  → 「次へ進む」をタップ
    → §5 の第2ダイアログ（意思確認・テキスト入力）を表示
    → 「削除する」をタップ
      → §6 の削除実行中 UI
      → 成功: ログアウト処理 → ログイン画面へ遷移
      → 失敗: エラー表示（再試行可能）
  → いずれかの段階で「キャンセル」 → settings/account へ戻る
```

### 4.3 ボタン仕様

| ボタン | スタイル | 挙動 |
|--------|---------|------|
| 「キャンセル」| テキストボタン / `colorTextSecondary` / 高さ 48pt | ダイアログを閉じる |
| 「削除に進む →」| アウトラインボタン（塗りつぶしなし）/ `colorError` ボーダー 1.5pt / `colorError` テキスト / 高さ 48pt / `radiusLg` | 第2ダイアログへ進む |

**「削除に進む」をアウトラインスタイルにする理由:** 完全な「塗りつぶし赤ボタン」は最終確認（ステップ 2）に取っておく。段階的に危険度を高める表現でユーザーが誤操作しにくくする。

### 4.4 プレミアム購読中ユーザーへの注意喚起

プレミアム購読が有効な場合のみ、削除データリストの下に追加で警告を表示する:

```
┌──────────────────────────────────────────────────────────────┐
│  [InfoIcon 16pt / colorWarning]                              │
│  プレミアムプランをご利用中の方へ                             │
│  textSm / colorTextPrimary / fontWeight: 600                 │
│                                                              │
│  アカウントを削除してもプレミアムプランは                     │
│  自動的には解約されません。                                   │
│  Google Play の定期購入管理ページから                         │
│  先にプランを解約されることをおすすめします。                  │
│  textSm / colorTextSecondary                                 │
│                                                              │
│  [Google Play 定期購入の管理 →]（テキストリンク）            │
│  colorTextLink                                               │
└──────────────────────────────────────────────────────────────┘
```

- 背景: `colorWarningBg`（`#fdf8e1`）/ 左端ボーダー 3pt `colorWarning` / 角丸 `radiusMd`
- 「Google Play 定期購入の管理」: `Linking.openURL()` で Google Play の定期購入管理ページを開く（具体的な URL: `https://play.google.com/store/account/subscriptions`）
- **store-compliance.md 要件:** Stripe 等の外部決済ページへの誘導は禁止。Google Play の管理ページへの誘導は許可されている

**購読状態の取得:** `isPremium`（サーバーから取得した購読状態。`lib/queries/` 経由）が true の場合のみこのブロックを表示する。

### 4.5 ダイアログのモーダル外形

- 背景オーバーレイ: `colorTextPrimary` 40% 不透明度（`rgba(26, 26, 26, 0.4)`）
- ダイアログコンテナ:
  - 背景: `colorBackground`（`#ffffff`）
  - 角丸: `radius2xl`（18pt）
  - 影: `shadow-washi-lg`
  - 水平マージン: `spacing6`（24pt）から各端
  - 内側パディング: `spacing6`（24pt）
- `accessibilityViewIsModal={true}` で背景へのフォーカスを防ぐ
- `accessibilityRole="alert"` / フォーカスをタイトルに移動させる

```
│                                                              │
│   確認のために「削除する」と入力してください。               │
│   textBase / colorTextSecondary / 中央揃え                   │
│                                                              │
│   ┌──────────────────────────────────────────────────────┐  │
│   │ 「削除する」と入力してください                       │  │
│   └──────────────────────────────────────────────────────┘  │
│   TextInput / 高さ 48pt / ボーダー colorBorder / radiusMd   │

│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 5.3 意思確認テキスト入力の仕様

- `placeholder`: 「削除する」と入力してください
- `placeholderTextColor`: `colorTextTertiary`
- `autoCapitalize: "none"` / `autoCorrect: false`
- `returnKeyType: "done"`
- 入力が「削除する」と完全一致した場合のみ「削除する」ボタンを活性化する
  - 前後の空白は無視する（`trim()` 後に比較）
  - 大文字・小文字は区別しない（全角・半角の違いは区別する — 全角で「削除する」と入力した場合も一致とみなす）
- フォーカス時ボーダー: 2pt `colorBorderFocus`
- 入力済みで不一致の場合: ボーダー `colorError` + インラインエラーテキスト「「削除する」と入力してください」

**意思確認入力を設ける理由:** 不可逆操作への誤タップを防ぐために「削除する」というテキスト入力を要求する。これは GitHub・Supabase 等の設計パターンに倣った確認方式である。

### 5.4 ボタン仕様

| ボタン | スタイル | 条件 |
|--------|---------|------|
| 「キャンセル」| テキストボタン / `colorTextSecondary` / 高さ 48pt | 常時タップ可 |
| 「削除する」（活性）| 背景 `colorError` / テキスト `colorTextInverse`（`#ffffff`）/ 高さ 48pt / `radiusLg` | 意思確認テキストが一致している |
| 「削除する」（非活性）| 背景 `colorSurfaceMuted` / テキスト `colorTextTertiary` / 高さ 48pt | テキストが一致していない |
| 「削除する」（送信中）| 背景 `colorError` / テキスト非表示 + スピナー（白）/ 高さ 48pt | API 呼び出し中 |

- `accessibilityRole="button"` / `accessibilityLabel="アカウントを完全に削除する"` / `accessibilityHint="この操作は取り消せません。"`
- `accessibilityState: { disabled }` を送信中・不一致時に設定する

### 5.5 「キャンセル」の挙動

- 第1ダイアログと第2ダイアログどちらの「キャンセル」も: 両ダイアログを閉じて `settings/account` に戻る
- 意思確認テキストの入力内容はリセットする

## 6. 削除実行中 UI

### 6.1 送信中の表示

「削除する」ボタン押下後:
- 「削除する」ボタンをスピナーに変更する
- 「キャンセル」ボタンを disabled にする（削除中の離脱を防ぐ）
- 意思確認テキスト入力を disabled にする
- ダイアログのオーバーレイをタップしても閉じられないようにする（`onRequestClose` を空実装にする）

### 6.2 処理フロー

```
「削除する」をタップ（意思確認テキスト一致後）
  → ダイアログの「削除する」ボタンをスピナーに変更
  → DELETE /api/v1/users/me（または POST /api/v1/users/me/delete）を呼び出す
    （想定エンドポイント。正本は OpenAPI — core に要確認）
  → 成功:
      §7 のログアウト処理を実行する
  → 失敗:
      ダイアログの「削除する」ボタンを re-enable する
      「キャンセル」ボタンを re-enable する
      §8 のエラー表示を行う
```

## 7. 削除成功後の処理

### 7.1 ログアウト処理（auth-tokens.md §ログアウト 準拠）

削除 API の成功後、以下を順に実行する:

1. Push 通知のデバイストークン登録を解除する（サーバーへの削除 API は失敗しても続行）
2. expo-secure-store からアクセストークン・リフレッシュトークンを削除する
3. `queryClient.clear()` でサーバーキャッシュを全消去する（前ユーザーのデータ残留防止）
4. 両ダイアログを閉じる
5. `router.replace(routes.login)` でログイン画面へ遷移する（スタックをすべて置き換え、戻れないようにする）

**サーバー呼び出し失敗時の fail-safe:** 削除 API が 5xx 等で失敗した場合でも、ローカルのトークン削除と画面遷移は必ず実施する（`auth-tokens.md` の fail-safe 方針に準拠）。ただし削除 API の失敗は Sentry に送信する。

### 7.2 完了後の画面

- `(auth)/login` 画面が表示される
- ログイン画面には削除完了の通知は表示しない（ユーザーがログイン状態でなくなるため）
- 代わりに: ダイアログを閉じる直前（ローカル処理実行中）にトースト「アカウントを削除しました」を短時間表示する（トーストは画面遷移後に消える）

## 8. エッジケース

### 8.1 ローディング

- `settings/account` の初期表示: メールアドレス・購読状態を取得する場合、取得中は各フィールドに `ScreenLoading`（spinner または スケルトン）を表示する
- プレミアム判定（`isPremium`）の取得中: 第1ダイアログのプレミアム警告ブロックを非表示にしておく（取得後に表示）

### 8.2 空状態

- この画面に「空状態」は存在しない（設定画面のため）

### 8.3 エラー

| エラー種別 | 表示箇所 | エラー定数カテゴリ |
|-----------|---------|-----------------|
| 削除 API 失敗（ネットワーク・5xx）| 第2ダイアログ内のインラインエラー（`FormErrorMessage` 相当）| `ERR_ACCOUNT_DELETE_FAILED` 相当 |
| 削除 API 失敗（403）| 第2ダイアログ内のインラインエラー | `ERR_FORBIDDEN` |
| 削除 API 失敗（429）| 第2ダイアログ内のインラインエラー | `ERR_RATE_LIMIT` |
| オフライン | 第2ダイアログ内のインラインエラー | `ERR_OFFLINE_ACTION` |

エラー発生時はダイアログを閉じずにその場にインラインエラーを表示し、ユーザーが再試行できる状態を維持する。

### 8.4 オフライン

- `OfflineBanner` を `settings/account` 画面上部に表示する
- ダイアログが表示中にオフラインになった場合: 「削除する」ボタンタップ時に検知し、ダイアログ内にインラインエラーを表示する

### 8.5 意思確認テキストの誤入力

- 「削除する」と完全一致しない間は「削除する」ボタンが非活性のため送信されない
- 入力済みで不一致の状態でフォーカスを外した場合: インラインエラー「「削除する」と入力してください」を表示する

### 8.6 削除処理中に画面を閉じようとした場合（iOS）

- iOS のモーダルスワイプダウンで閉じようとしても、削除処理中は `onRequestClose` を無効にして閉じられないようにする
- Android のハードウェアバックボタンも同様に無効化する（Expo Router の `BackHandler` で制御）

## 11. データの流れ

### 11.1 削除 API

- `DELETE /api/v1/users/me`（または `POST /api/v1/users/me/delete`）

### 11.2 必要なデータ

| データ | 取得元 | 用途 |
|--------|-------|------|

## 12. store-compliance.md との対応確認

| 要件 | 本仕様での対応 |
|------|-------------|
| アプリ内からのアカウント削除導線（必須）| `settings/account` の「危険ゾーン」セクションに「アカウントを削除する」を配置 |
| 削除が不可逆であることの明示 | 第1ダイアログでの警告文言 + 削除されるデータのリスト |
| 二段階確認 | 第1（警告）→ 第2（意思確認テキスト入力）の 2 ステップ |
| 購読中ユーザーへの案内 | プレミアム購読中は第1ダイアログに `PremiumWarningBlock` を表示（Google Play 定期購入管理へ誘導） |
| 外部決済誘導の禁止 | Stripe 等の外部決済ページへのリンクは一切出さない |
| Play Console データ削除申告 | Play Console の「データセーフティ」でデータ削除手段を申告する必要がある（**PM / ストア登録担当への申し送り**）|
