## 3. 機能インベントリ（データ出所分類）

| 機能 | データ出所分類 | 既存 API エンドポイント | 備考 |
|-----|-------------|----------------------|------|
| 病害虫一覧（検索・カテゴリ・体長フィルタ） | 既存 API | `GET /api/v1/pesticides/disease-pests?category=&search=&bodySizeMm=` | 既存実装あり |
| 病害虫詳細 | 既存 API | `GET /api/v1/pesticides/disease-pests/{slug}` | 既存実装あり |
| 農薬製品一覧（検索・タイプ・病害虫 ID・剤型コードフィルタ） | 既存 API | `GET /api/v1/pesticides/products?search=&type=&diseasePestId=&formulationTypeCode=` | 既存実装あり |
| 農薬製品詳細 | 既存 API | `GET /api/v1/pesticides/products/{slug}` | 既存実装あり |
| 有効成分一覧（検索） | 既存 API | `GET /api/v1/pesticides/ingredients?search=` | 既存実装あり |
| 有効成分詳細 | 既存 API | `GET /api/v1/pesticides/ingredients/{slug}` | 既存実装あり |
| 展着剤タイプ一覧 + 各タイプの製品リスト | **新 API 必要** | 未定義 | `SpreaderType` モデル + 紐付く製品を返すエンドポイントが必要 |
| 展着剤製品一覧（spreaderTypes を持つ製品の一覧） | **新 API 必要** | 未定義 | `GET /api/v1/pesticides/products?type=spreader` 相当だが現行の type パラメータは `PesticideType` enum に限定されており spreader は対象外 |
| 展着剤タイプ詳細（概要・効果・利用注意） | **新 API 必要** | 未定義 | `GET /api/v1/pesticides/spreaders/{slug}` |
| コラム一覧 | **新 API 必要** | 未定義 | `PesticideColumn` モデルの一覧 |
| コラム記事詳細 | **新 API 必要** | 未定義 | `GET /api/v1/pesticides/columns/{slug}` |
| 剤型一覧（剤型名・コード・製品件数） | **新 API 必要** | 未定義 | `FormulationType` モデルの一覧 |
| 混用チェッカー（農薬全件 + 混用不可ペアリスト） | **新 API 必要** | 未定義 | `GET /api/v1/pesticides/mixing-data`（全農薬リスト + 混用不可テーブル全件）|
| 希釈計算ツール | クライアント計算 | 不要 | 純粋数値計算。`水量 ÷ 希釈倍率 = 薬剤量`。定数は `lib/constants/` に定義 |
| 散布方法ガイド | 静的コンテンツ | 不要 | 完全にハードコードされた説明文と希釈対応表（SPRAY_DILUTION_RATIOS・SPRAY_WATER_VOLUMES_ML 定数を使用） |

### サーバー新 API まとめ（core に要確認）

以下の 6 エンドポイントが新規に必要。いずれも Bon_Log_cfw リポジトリ側で実装が必要。

1. `GET /api/v1/pesticides/spreaders` — 展着剤タイプ一覧（id, slug, name, description, sortOrder, 紐付く製品の id/slug/name/description/formulationType）
2. `GET /api/v1/pesticides/spreaders/{slug}` — 展着剤タイプ詳細（name, description, effect, usageNote, 紐付く製品リスト）または製品 slug の場合は製品詳細（products/{slug} への 301 相当案内）
3. `GET /api/v1/pesticides/spreader-products` — spreaderTypes を 1 件以上持つ農薬製品の一覧（id, slug, name, formulationType, registrationNumber, spreaderTypes）
4. `GET /api/v1/pesticides/columns` — コラム一覧（id, slug, title, category, publishedAt, sortOrder）
5. `GET /api/v1/pesticides/columns/{slug}` — コラム詳細（title, content, category, publishedAt, createdAt, updatedAt）
6. `GET /api/v1/pesticides/mixing-data` — 混用チェッカー用データ一括（pesticides: \[id, name, slug, pesticideType\]、incompatibilities: \[pesticideId, incompatibleWithId\]）

また剤型は既存の `formulationTypeCode` フィルタが製品一覧 API に存在するが、剤型マスタ一覧（`FormulationType` の一覧）を返すエンドポイントが未定義のため以下も必要:

7. `GET /api/v1/pesticides/formulations` — 剤型マスタ一覧（id, code, name, description, sortOrder, pesticides_count）

#### 検索フォームの挙動

- テキスト入力 + 検索ボタン: `search` パラメータで製品一覧 API を呼ぶ
- タイプチップタップ:
  - `全て`: フィルタなしリセット → あいうえお順全一覧表示
  - `害虫 / 病気 / 益虫`: `category` パラメータで DiseasePestGrid に切り替え
  - `殺虫剤 / 殺菌剤 / 殺ダニ剤 / 複合剤`: `type` パラメータで農薬製品絞り込み
  - `展着剤`: 展着剤製品リスト（SpreaderResults）に切り替え

#### 計算ロジック

- 正モード（薬剤量計算）: `薬剤量（mL）= 水量（mL）÷ 希釈倍率`
