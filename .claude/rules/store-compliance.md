# ストア審査・コンプライアンスルール

モバイル固有の新設ルール。App Store / Google Play の審査要件は機能要件と同格に扱う（計画書 Phase 4）。

## 課金（App Store Review Guideline 3.1.1）

- プレミアム購読などデジタル機能の販売は **IAP / Play Billing のみ**
- **Stripe 決済・外部決済ページへの誘導をアプリ内に出さない**（ボタン・リンク・文言すべて）
- 解約・購読管理は OS の購読管理画面へ誘導する

## UGC 要件（App Store Review Guideline 1.2）— MVP 必須

- **通報（コンテンツ・ユーザー）とブロックの UI をモバイルで露出する**
  （Web に機能が存在しても、アプリ内から実行できなければ審査に通らない — 計画書 Phase 4）
- 利用規約への同意導線とモデレーション体制の明示
- ブロック・ミュートがフィード・通知に即時反映されること（サーバー側のフィルタは既存実装を共有）

## アカウント

- アカウント作成があるアプリは**アプリ内からのアカウント削除導線が必須**（App Store Review 5.1.1(v)）
- Google ログイン等のサードパーティログインを出す場合、iOS では **Sign in with Apple の併設が原則必要**（Guideline 4.8）。
  対応方針（Apple ログイン追加 or メール+Google のみで審査リスクを取るか）を Phase 4 までに決定する
  — 計画書未記載のため要判断事項として扱う

## プライバシー

- iOS: プライバシーマニフェスト（required reason API / 収集データ申告）を整備（計画書 Phase 4 成果物）
- App Store「App のプライバシー」/ Play「データセーフティ」の申告を実際のデータ収集（Sentry・RevenueCat 等）と一致させる
- トラッキングに該当する SDK 挙動を入れる場合は ATT 許可が必要 — 既定では導入しない

## 申請前チェックリスト（Phase 4 受け入れ基準と連動）

- [ ] サンドボックス購入 → RevenueCat Webhook → DB 反映 → プレミアム機能解放が通る
- [ ] 通報・ブロックがアプリ内から実行できる
- [ ] アカウント削除導線がある
- [ ] プライバシーマニフェスト / データセーフティ申告が実態と一致
- [ ] 外部決済への誘導が存在しない
- [ ] 審査用デモアカウントを準備済み
